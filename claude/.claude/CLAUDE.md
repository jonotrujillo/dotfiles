# Personal Claude Code Preferences

## Workflow

- Show diffs before applying changes to existing files
- When multiple approaches exist, present options before implementing
- When fixing a bug, explain root cause before writing the fix
- Before starting unrelated work mid-task, stop and ask whether to handle it separately
- Suggest a new branch when a sidetrack touches different concerns than the current task

## Commits

- No co-author attribution in commit messages
- Prefer small focused PRs over large ones
